# Data Model: Silver Tier Features

## Email Entity
**Fields**:
- id (string): Unique identifier from Gmail API
- sender (string): Email address of sender
- subject (string): Subject line of email
- date_received (datetime): Timestamp when email was received
- body_snippet (string): First 500 characters of email content
- status (enum): Current processing state (unread, needs_action, pending_approval, approved, done)
- importance_level (enum): Importance categorization (low, medium, high, critical)
- raw_data (dict): Original email data from Gmail API

**Validation Rules**:
- sender must be valid email format
- date_received must be in past
- status must be one of defined enum values

**State Transitions**:
- unread → needs_action (when processed by Gmail watcher)
- needs_action → pending_approval (when AI generates reply)
- pending_approval → approved (when human approves)
- approved → done (when email sent)
- pending_approval → needs_action (when human rejects)

## LinkedIn Post Entity
**Fields**:
- id (string): Unique identifier
- content (string): Generated post content
- topic (string): Subject/theme of the post
- date_created (datetime): Timestamp when post was generated
- date_approved (datetime): Timestamp when post was approved (null if not approved)
- date_posted (datetime): Timestamp when post was published (null if not posted)
- status (enum): Processing state (draft, pending_approval, approved, posted)
- source_topic (string): Original topic that triggered generation

**Validation Rules**:
- content length must be within LinkedIn character limits
- topic must not be empty
- date_created must be in past
- status must be one of defined enum values

**State Transitions**:
- draft → pending_approval (when generated by AI)
- pending_approval → approved (when human approves)
- approved → posted (when published to LinkedIn)
- pending_approval → draft (when human rejects)

## Approval Entity
**Fields**:
- id (string): Unique identifier
- action_type (enum): Type of action (email_reply, linkedin_post)
- content (string): Content to be approved
- date_submitted (datetime): Timestamp when submitted for approval
- date_approved (datetime): Timestamp when approved (null if not approved)
- date_rejected (datetime): Timestamp when rejected (null if not rejected)
- status (enum): Approval state (pending, approved, rejected)
- reason_for_rejection (string): Reason for rejection (if applicable)

**Validation Rules**:
- action_type must be one of defined enum values
- content must not be empty
- date_submitted must be in past
- status must be one of defined enum values
- reason_for_rejection must be provided if status is rejected

**State Transitions**:
- pending (when created)
- approved (when human approves)
- rejected (when human rejects)

## API Credential Entity
**Fields**:
- service_type (enum): Service name (gmail, linkedin, whatsapp)
- client_id (string): OAuth client ID
- client_secret (string): OAuth client secret
- access_token (string): Current access token
- refresh_token (string): Refresh token for OAuth
- token_expiry (datetime): When access token expires
- scopes (array): Granted OAuth scopes

**Validation Rules**:
- service_type must be one of defined enum values
- client_id and client_secret must not be empty
- tokens must be properly formatted
- token_expiry must be in future when valid

## Rate Limit Entity
**Fields**:
- service_type (enum): Service name (gmail, linkedin, email_sending)
- limit_count (integer): Maximum allowed per period
- current_count (integer): Current usage in current period
- period_start (datetime): When current period started
- last_reset (datetime): When count was last reset

**Validation Rules**:
- limit_count must be positive integer
- current_count must be non-negative integer
- current_count must not exceed limit_count
- period_start and last_reset must be in past

**State Transitions**:
- Resets to 0 when period_start + period_duration exceeds current time
